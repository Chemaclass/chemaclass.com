# No em dashes

Never use the em dash character (`—`, U+2014) anywhere: blog posts, readings, talks, slide decks, speaker notes, commit messages, PR descriptions, or any generated text.

Also avoid the en dash (`–`, U+2013) in prose.

Use instead:
- A period for two separate statements.
- A comma for a brief aside.
- A colon for a lead-in or definition.
- Parentheses for a true aside.

This matches the site's writing style: plain text, clear storyline, no em dashes.
